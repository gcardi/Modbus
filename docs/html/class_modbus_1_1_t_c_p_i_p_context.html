Detailed Description

Modbus context extended with an explicit MBAP transaction identifier.

In Modbus TCP/UDP the MBAP header contains a two-byte Transaction Identifier that the master sets and the slave echoes back. TCPIPContext stores this value so that TCPIPProtocol can validate that a response corresponds to the request that was sent.

The transaction identifier is typically a monotonically increasing sequence number managed by the caller.
